## 项目概述

这是一个基于 **librtmp** 和 **FFmpeg** 的 **RTMP 推流应用**，用 C++ 和 Qt 开发。

### 主要功能
- **采集**：音频（麦克风/PCM文件）和视频（桌面/YUV文件）
- **编码**：H.264 视频编码 + AAC 音频编码
- **推流**：通过 RTMP 协议推送到直播服务器
- **显示**：实时显示推流画面（基于SDL）
- **同步**：音视频同步支持

### 开发环境
- **语言**：C++11
- **编译器**：MSVC 或 MinGW
- **依赖**：FFmpeg 4.2.1, librtmp, SDL2, Qt

### 整体架构图

```
┌─────────────────────────────────────────────────────┐
│                  PushWork (推流工作流)                │
│            (统筹和协调各个模块的工作)                │
└──┬──────────────────────┬──────────────────────┬────┘
   │                      │                      │
   ▼                      ▼                      ▼
┌────────────────┐  ┌────────────────┐  ┌──────────────────┐
│  音频采集链     │  │  视频采集链     │  │  RTMP 推流链      │
├────────────────┤  ├────────────────┤  ├──────────────────┤
│ AudioCapturer  │  │ VideoCapturer  │  │  RTMPPusher      │
│    (采集PCM)   │  │    (采集YUV)   │  │   (异步推流)     │
│      ↓         │  │      ↓         │  │                  │
│AudioResampler  │  │  H264Encoder   │  │ 发送metadata     │

│  AACEncoder    │  │  NaluLoop      │  │ 发送H.264 packet │
│  (编码AAC)     │  │ (NALU队列)     │  │ 发送AAC packet   │

```

## 模块详细说明

### A. 音频处理模块

#### 1. **AudioCapturer** (`audiocapturer.h/cpp`)
- **职责**：采集音频数据
- **输入**：麦克风或PCM文件
- **输出**：原始PCM数据
- **关键方法**：
  - `Init(const Properties& properties)`：初始化采集器
  - `Loop()`：线程循环，持续采集数据
  - `AddCallback()`：注册数据回调
- **支持模式**：
  - 测试模式：从文件读取 PCM 数据
  - 实时模式：从麦克风采集（未完全实现）

**配置参数**：
```cpp
"audio_test": 1                           // 测试模式开关
"input_pcm_name": "buweishui.pcm"        // 测试文件路径
"mic_sample_rate": 48000                 // 采样率
"mic_channels": 2                         // 通道数
"mic_sample_fmt": AV_SAMPLE_FMT_S16      // 采样格式
```

#### 2. **AudioResampler** (`audioresample.h/cpp`)
- **职责**：重采样音频数据
- **输入**：原始 PCM 数据
- **输出**：重采样后的 PCM 数据
- **用途**：将采集的音频转换为编码器所需的格式
- **关键方法**：
  - `Init()`：初始化重采样器
  - `Resample()`：执行重采样

#### 3. **AACEncoder** (`aacencoder.h/cpp`)
- **职责**：AAC 音频编码
- **输入**：PCM 数据
- **输出**：AAC 编码数据（带 ADTS 头）
- **编码标准**：AAC-LC (Low Complexity)
- **关键方法**：
  - `Init(const Properties& properties)`：初始化编码器
  - `Encode(AVFrame *frame, uint8_t* out, int out_len)`：编码一帧
  - `GetAdtsHeader()`：生成 ADTS 头信息

**配置参数**：
```cpp
"audio_sample_rate": 48000       // 编码采样率
"audio_channels": 2               // 编码通道数
"audio_bitrate": 64*1024          // 编码码率 (bps)
```

#### 4. **AACRtmpPackager** (`aacrtmppackager.h/cpp`)
- **职责**：将 AAC 数据打包成 RTMP 格式
- **输入**：AAC 编码数据
- **输出**：RTMP 音频包

### B. 视频处理模块

#### 1. **VideoCapturer** (`videocapturer.h/cpp`)
- **职责**：采集视频数据
- **输入**：桌面或 YUV 文件
- **输出**：原始 YUV 数据
- **关键方法**：
  - `Init(const Properties& properties)`：初始化采集器
  - `Loop()`：线程循环，持续采集帧
  - `AddCallback()`：注册数据回调
- **支持模式**：
  - 测试模式：从 YUV420p 文件读取
  - 实时模式：从桌面截图（未完全实现）

**配置参数**：
```cpp
"video_test": 1                           // 测试模式开关
"input_yuv_name": "720x480_25fps.yuv"   // 测试文件路径
"desktop_x": 0, "desktop_y": 0          // 采集区域起点
"desktop_width": 720                     // 采集宽度
"desktop_height": 480                    // 采集高度
"desktop_fps": 25                        // 采集帧率
"desktop_pixel_format": AV_PIX_FMT_YUV420P  // 像素格式
```

#### 2. **H264Encoder** (`h264encoder.h/cpp`)
- **职责**：H.264 视频编码
- **输入**：YUV 视频帧
- **输出**：H.264 NALU 单元（包含 SPS、PPS、I 帧、P 帧）
- **编码配置**：
  - Profile: Baseline / Main / High
  - 支持 I 帧间隔 (GOP)
  - 支持 B 帧设置
- **关键方法**：
  - `Init(const Properties& properties)`：初始化编码器
  - `Encode(AVFrame *frame, uint8_t* out, int &out_size)`：编码
  - `get_sps()` / `get_pps()`：获取序列头参数

**配置参数**：
```cpp
"video_width": 720                       // 编码宽度
"video_height": 480                      // 编码高度
"video_fps": 25                          // 编码帧率
"video_gop": 25                          // I 帧间隔
"video_bitrate": 512*1024                // 编码码率 (bps)
"video_b_frames": 0                      // B 帧数量（直播通常为0）
```

#### 3. **NaluLoop** (`naluloop.h/cpp`)
- **职责**：管理 H.264 NALU 队列
- **输入**：编码后的 NALU 单元
- **输出**：NALU 队列供 RTMP 发送
- **作用**：缓冲和调度 NALU 单元

#### 4. **ImageScale** (`imagescale.h/cpp`)
- **职责**：图像缩放
- **用途**：调整视频分辨率

#### 5. **VideoOutSDL** (`videooutsdl.h/cpp`)
- **职责**：使用 SDL 显示视频
- **用途**：实时显示推流画面（调试对比）
- **依赖**：SDL2 库

### D. RTMP 推流模块

#### 1. **RTMPPusher** (`rtmppusher.h/cpp`)
- **职责**：异步 RTMP 推流
- **继承自**：`NaluLoop` 和 `RTMPBase`
- **关键功能**：
  - 发送 FLV metadata
  - 发送 Audio Specific Config（AAC 配置）
  - 发送 Video Sequence Header（H.264 配置）
  - 发送 H.264 视频数据包
  - 发送 AAC 音频数据包
- **关键方法**：
  - `SendMetadata(FLVMetadataMsg *metadata)`
  - `SendAudioSpecificConfig(char* data, int length)`
  - `sendH264SequenceHeader(VideoSequenceHeaderMsg *seq_header)`
  - `sendH264Packet(char *data, int size, bool is_keyframe, unsigned int timestamp)`

**推流顺序**：
1. 连接 RTMP 服务器
2. 发送 Metadata
3. 发送 Video Sequence Header (SPS + PPS)
4. 发送 Audio Specific Config
5. 循环发送音视频数据包

#### 2. **RTMPBase** (`rtmpbase.h/cpp`)
- **职责**：RTMP 基础连接管理
- **用途**：
  - 连接/断开 RTMP 服务器
  - 管理 RTMP 上下文
  - 发送原始 RTMP 包

#### 3. **RtmpPackager** (`rtmppackager.h`)
- **职责**：RTMP 数据打包接口
- **用途**：定义数据打包规范

### E. 工作流协调模块

#### 1. **PushWork** (`pushwork.h/cpp`) ⭐ 核心类
- **职责**：统筹和协调所有模块
- **核心职责**：
  - 初始化所有子模块（采集、编码、推流）
  - 启动各个线程
  - 处理各模块的回调
  - 协调音视频数据流
- **关键方法**：
  - `Init(const Properties& properties)`：初始化
  - `DeInit()`：反初始化
  - `Loop()`：主工作循环
  - `AudioCallback(NaluStruct* nalu_data)`：处理编码后的 AAC 数据
  - `VideoCallback(NaluStruct* nalu_data)`：处理编码后的 H.264 数据
  - `PcmCallback(uint8_t* pcm, int32_t size)`：处理 PCM 数据
  - `YuvCallback(uint8_t* yuv, int32_t size)`：处理 YUV 数据

**工作流程**：
```
Init():
  ├─ 创建 AudioCapturer
  ├─ 创建 AudioResampler
  ├─ 创建 AACEncoder
  ├─ 创建 VideoCapturer
  ├─ 创建 H264Encoder
  ├─ 创建 RTMPPusher
  ├─ 启动各个采集线程
  └─ 注册回调函数

Loop():
  └─ 保持运行直到推流结束

AudioCallback():
  └─ 接收编码后的 AAC 数据 → 发送给 RTMPPusher

VideoCallback():
  └─ 接收编码后的 H.264 NALU → 发送给 RTMPPusher
```

#### 2. **MediaBase** (`mediabase.h`) ⭐ 基础类

#### 3. **CommonLooper** (`commonlooper.h/cpp`)

- **继承者**：`AudioCapturer`、`VideoCapturer`

#### 4. **Looper** (`looper.h/cpp`)

## 数据流向

### 完整的数据流图

```
PCM 文件输入                          YUV 文件输入
    │                                     │
    ▼                                     ▼
AudioCapturer                      VideoCapturer
  (PCM 采集)                         (YUV 采集)
    │                                     │
    ▼                                     ▼
AudioResampler                           │
(重采样到目标格式)                       │
    │                                     │
    ▼                                     ▼
AACEncoder                         H264Encoder
(AAC 编码)                          (H.264 编码)
    │                                     │
    └──────────┬──────────────────────────┘
               │
               ▼ (回调)
            PushWork
               │
    ┌──────────┴──────────┐
    │                     │
    ▼                     ▼
AudioCallback         VideoCallback
(处理 AAC 数据)       (处理 H.264 NALU)
    │                     │
    └──────────┬──────────┘
               │
               ▼
            RTMPPusher
     (打包 RTMP 数据包)
               │
               ▼
          RTMP 服务器
        (直播平台)
```

### 关键时间点

1. **采集阶段**：
   - AudioCapturer / VideoCapturer 在各自的线程中持续采集
   - 按时间间隔采集（相同的帧率）

2. **编码阶段**：
   - 编码器实时编码采集到的数据
   - 编码后的数据通过回调返回给 PushWork

3. **推送阶段**：
   - 需要发送 Metadata → Video Sequence Header → Audio Specific Config
   - 然后循环发送音视频数据包
   - 需要匹配时间戳保证同步

## 关键技术点

### FFmpeg 编码 API

**H.264 编码流程**：
```cpp
1. av_codec_find_encoder()        // 查找编码器
2. avcodec_alloc_context3()       // 分配上下文
3. av_opt_set()                   // 设置编码参数
4. avcodec_open2()                // 打开编码器
5. av_frame_alloc()               // 分配 Frame
6. avcodec_send_frame()           // 送入数据
7. avcodec_receive_packet()       // 接收编码包
```

**AAC 编码流程**：
```cpp
类似 H.264，但输入是 PCM 数据
- av_frame_get_buffer()  // 为 Frame 分配 PCM 缓冲
- av_frame_make_writable()
```

### RTMP 协议基础

**RTMP 包结构**：
```
RTMP Header (12 bytes)
├─ Timestamp (3 bytes)
├─ Message Length (3 bytes)
├─ Message Type (1 byte)
└─ Stream ID (4 bytes)

RTMP Payload
├─ Metadata
├─ Audio/Video Config
└─ Audio/Video Data
```

**关键消息类型**：
- `0x12` (18): Notify (metadata)
- `0x08` (8): Audio data
- `0x09` (9): Video data

### 音视频同步方案

**时间戳管理**：
1. 采集时记录系统时间
2. 根据采样率计算 PTS
3. 所有数据包使用统一的时间基准
4. RTMP 包中的时间戳需要从第一包开始计时

## 常见问题

### Q1: 为什么有这么多模块？
**A**: 关注点分离。每个模块负责一个明确的功能：
- 采集层负责采集
- 编码层负责编码
- 推流层负责网络传输
- 协调层负责管理

### Q2: 回调是怎么工作的？
**A**: 采集和编码线程是独立的：
1. AudioCapturer 采集 PCM
2. AudioResampler 重采样
3. AACEncoder 编码后调用 AudioCallback
4. AudioCallback 被 PushWork 处理
5. PushWork 将数据发给 RTMPPusher

### Q3: 如何自己写编码逻辑？
**A**: 参考 AACEncoder 和 H264Encoder：
1. 学会 FFmpeg 的基本 API
2. 配置编码参数
3. 在 Init() 中初始化
4. 在 Encode() 中执行编码

### Q4: 如何添加自己的视频源？
**A**: 修改 VideoCapturer：
1. 在 Init() 中识别你的视频源
2. 在 Loop() 中读取数据
3. 回调通知 PushWork

### Q5: 如何调试时间戳问题？
**A**: 
1. 在 PushWork 中添加日志打印 PTS
2. 对比输入和输出的时间戳
3. 检查采样率和帧率的配置
